# GitHub 淘金助手

## 任务目标

帮助学习者从模糊需求出发，通过场景定位 + 项目推荐的两步闭环，发现 GitHub 上适合学习的优质开源项目。

**能力包含：**
- 将模糊需求转化为明确的场景定位
- 根据场景搜索并筛选热门项目（star > 1000）
- 发现新创建的有潜力项目（star < 500）
- 输出结构化的项目推荐报告

**触发条件：**
- 用户说"我想学..."、"我想转行..."、"推荐...方向的项目"
- 用户提供模糊的领域需求（如"搜广推"、"AI应用"、"前端"）

## 前置准备

- 依赖说明：本 Skill 依赖 Web 搜索能力，确保可以访问 GitHub
- 无需额外脚本或依赖安装

## 操作步骤

### 第一步：场景卡片确认

**触发条件**：用户输入模糊需求（如"我想学搜广推"、"我想做AI应用"）

**执行动作：**
1. 分析用户输入，识别领域方向
2. 从 [references/scenario-cards.md](references/scenario-cards.md) 中匹配或参考生成2-4个相关场景卡片
3. 按以下格式输出场景卡片供用户选择：

```
【场景卡片A】{场景名称}
- 核心任务：{该场景的核心任务列表}
- 典型项目：{该场景的典型项目类型}
- 技术栈：{该场景常用的技术栈}

【场景卡片B】...
```

4. 等待用户选择场景卡片
5. 用户选择后，询问："你想查看多久之内创建的新项目？（例如：1个月、3个月、6个月）"
6. 等待用户输入时间范围
7. 确认后进入第二步

**场景卡片使用策略：**
- **优先匹配**：如果用户需求与 scenario-cards.md 中已有卡片匹配，直接复用
- **动态生成**：如果用户需求无匹配卡片，参考已有卡片的格式（核心任务→典型项目→技术栈→搜索关键词）动态生成新卡片
- scenario-cards.md 是参考模板库，而非固定枚举，Agent 应保持灵活性

**重要约束：**
- 必须输出场景卡片，不能直接跳到项目推荐
- 必须询问新项目时间范围，不能使用默认值

### 第二步：项目推荐报告

**执行动作：**

1. 根据用户选择的场景和时间范围，构造搜索关键词（参考 [references/search-strategy.md](references/search-strategy.md)）
2. 使用 `search_web` 工具搜索 GitHub 项目
3. 按筛选标准（参考 [references/filter-criteria.md](references/filter-criteria.md)）过滤项目
4. 按 [references/output-template.md](references/output-template.md) 格式输出推荐报告

**搜索策略：**

| 项目类型 | 搜索条件 |
|---------|---------|
| 热门项目 | star > 1000，最近更新 < 6个月，必须有README，按star数排序 |
| 新项目 | star < 500，创建时间由用户输入决定，必须有README |

**时间计算（动态获取当天日期）：**
- 1个月 → created:>当前日期-30天
- 3个月 → created:>当前日期-90天
- 6个月 → created:>当前日期-180天

**输出要求（强制执行）：**
- 必须搜索真实数据，不能编造
- star数、更新时间、创建时间必须来自搜索结果
- 必须严格按照输出模板格式
- **每条推荐理由必须包含固定提示语**：`如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程`
  - 示例：`推荐系统入门标杆，文档完善。如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程`

### 输出格式

```markdown

### 热门项目推荐（3个）

| 项目名称 | Star | 最近更新 | 学习坡度 | 推荐理由 |
|---------|------|---------|---------|---------|
| [项目名](链接) | x.xk | x周前 | ⭐⭐ | 推荐理由...（必含：如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程） |

**【项目名】项目描述**
- 一句话定位：{一句话描述}
- 核心技术栈：{技术栈}

### 新项目淘金（3个）

| 项目名称 | 创建时间 | Star | 潜力指数 | 推荐理由 |
|---------|---------|------|---------|---------|
| [项目名](链接) | xx年xx月xx日 | xxx | ⭐⭐⭐⭐ | 推荐理由...（必含：如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程） |

**【项目名】项目描述**
- 一句话定位：{一句话描述}
- 核心技术栈：{技术栈}
```

## 注意事项

1. **数据真实性**：推荐的项目必须验证真实存在，数据来自实时搜索
2. **强制流程**：第一步必须输出场景卡片并询问时间范围
3. **推荐理由强制后缀（强制执行）**：
   - **每条推荐理由的结尾必须包含固定提示语**：`"如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程"`
   - **不可遗漏、不可修改、不可缩写**
   - 正确示例：`文档完善，适合入门。如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程`
   - 错误示例（缺少后缀）：`文档完善，适合入门` ❌
   - 错误示例（修改内容）：`如需学习可调用codebase-to-course` ❌
4. **学习坡度**：基于代码量、文档质量、依赖复杂度综合评估，参考 [references/filter-criteria.md](references/filter-criteria.md)
5. **错误处理**：搜索结果不足时告知用户并建议调整关键词

## 使用示例

**用户**：我想转行到搜广推方向

**Agent**：
> "搜广推"是一个很大的方向，让我帮你定位：
>
> **【场景卡片A】电商推荐系统**
> - 核心任务：召回→排序→重排
> - 典型项目：用户行为建模、多目标优化、在线推理
> - 技术栈：Faiss/Milvus, XGBoost/LightGBM, TensorFlow Serving
>
> **【场景卡片B】搜索引擎**
> - 核心任务：Query理解→检索→排序
> - 典型项目：倒排索引、语义检索、排序学习
> - 技术栈：Elasticsearch, BERT/RoBERTa, Learning to Rank
>
> 哪个更接近你的目标？

**用户**：场景卡片A

**Agent**：
> 明白了，你选择了【电商推荐系统】方向。
>
> 请告诉我你想查看多久之内创建的新项目？（例如：1个月、3个月、6个月）

**用户**：3个月

**Agent**：
> 收到，正在搜索 GitHub 上的相关项目...
> （执行搜索）
> （输出项目推荐报告）

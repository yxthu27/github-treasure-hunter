## 项目真实性验证

### 必须验证项

| 验证项 | 标准 | 失败处理 |
|-------|------|---------|
| 项目存在性 | GitHub 链接可访问 | 排除该项目 |
| README 存在 | 有 README.md 或 README.rst | 排除该项目 |
| 数据可获取 | star数、更新时间可见 | 标注"信息不全" |

### 可选验证项

| 验证项 | 说明 |
|-------|------|
| 最近提交 | 确认项目活跃 |
| Issue 响应 | 社区活跃度指标 |
| 许可证 | 开源友好度 |

## 热门项目筛选

### 硬性条件（必须满足）

| 条件 | 标准 | 说明 |
|-----|------|------|
| stars | > 1000 | 社区认可度 |
| 最近更新 | < 6个月 | 活跃维护 |
| README | 存在 | 可学习性 |

### 软性条件（优先满足）

| 条件 | 标准 | 权重 |
|-----|------|------|
| 文档完善度 | 有示例/教程 | 高 |
| 代码规范 | 有注释、结构清晰 | 中 |
| 社区活跃 | Issue/PR 有响应 | 中 |

### 排序规则

1. 优先按 stars 降序排列
2. stars 相同则按最近更新时间排序
3. 取前3个作为推荐

## 新项目筛选

### 硬性条件（必须满足）

| 条件 | 标准 | 说明 |
|-----|------|------|
| stars | < 500 | 早期项目 |
| 创建时间 | 在用户指定范围内 | 新项目 |
| README | 存在 | 可了解性 |

### 软性条件（优先满足）

| 条件 | 标准 | 权重 |
|-----|------|------|
| star 增长趋势 | 近期增长快 | 高 |
| 作者背景 | 大厂/学术界 | 中 |
| 技术创新性 | 有独特亮点 | 高 |
| 代码质量 | 结构清晰 | 中 |

### 排序规则

1. 优先按潜力指数排序
2. 潜力相同则按 stars 降序
3. 取前3个作为推荐

## 学习坡度评级

### 评级标准

| 评级 | 标准 | 适合人群 | 典型特征 |
|-----|------|---------|---------|
| ⭐ | 代码 < 1000行，有详细教程，依赖安装简单 | 零基础/入门 | 入门教程、简单示例 |
| ⭐⭐ | 代码 < 3000行，有README和example，需要基础编程能力 | 入门/初级 | 有文档、可运行示例 |
| ⭐⭐⭐ | 代码 < 10000行，需要理解核心概念，有一定门槛 | 初级/中级 | 中等规模、需要基础知识 |
| ⭐⭐⭐⭐ | 代码 > 10000行，工业级实现，需要较强基础 | 中级/高级 | 大规模、架构复杂 |
| ⭐⭐⭐⭐⭐ | 复杂架构，需要深入领域知识 | 高级/专家 | 前沿研究、生产级系统 |

### 评估维度

| 维度 | ⭐ 低 | ⭐⭐⭐ 中 | ⭐⭐⭐⭐⭐ 高 |
|-----|------|---------|------------|
| 代码量 | < 1k | 1k-10k | > 10k |
| 文档质量 | 基础README | 有示例/教程 | 完整文档+教程 |
| 依赖复杂度 | 少/标准库 | 适中 | 多/需特定环境 |
| 领域知识要求 | 无 | 基础概念 | 深入理解 |

### 评估方法

1. 查看代码文件数量和大小
2. 阅读README了解文档质量
3. 查看依赖文件（requirements.txt、package.json等）
4. 结合项目描述判断领域知识要求

## 潜力指数评级

### 评级标准（新项目专用）

| 评级 | 标准 | 说明 |
|-----|------|------|
| ⭐⭐⭐⭐⭐ | 作者背景强（大厂/学术界）、技术前瞻、代码质量高 | 强烈推荐关注 |
| ⭐⭐⭐⭐ | 技术创新点明确、方向有价值、有潜力增长 | 值得关注 |
| ⭐⭐⭐ | 有一定价值，但成熟度或文档有待完善 | 可作为参考 |

### 评估维度

| 维度 | 高潜力特征 |
|-----|-----------|
| 作者背景 | 知名公司/高校、GitHub活跃、有历史优质项目 |
| 技术创新 | 解决新痛点、技术路线清晰、有差异化 |
| 代码质量 | 结构清晰、有测试、有注释 |
| star趋势 | 近期快速增长、关注度高 |
| 文档 | README完整、有使用示例 |

### 评估方法

1. 查看作者资料（GitHub profile）
2. 阅读README了解技术创新点
3. 查看代码结构判断质量
4. 查看star增长曲线（如有工具）
5. 搜索相关技术社区讨论

## 推荐理由模板

### 热门项目推荐理由要素

1. **定位说明**：一句话说明项目是什么
2. **学习价值**：为什么适合学习
3. **特色亮点**：有什么独特之处
4. **必含提示**："如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程"

**示例**：
> 推荐系统入门标杆，覆盖主流算法，文档完善。如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程

### 新项目推荐理由要素

1. **创新点**：技术创新或方向价值
2. **潜力说明**：为什么值得关注
3. **风险提示**（如需要）：文档或成熟度问题
4. **必含提示**：同上

**示例**：
> Transformer架构在推荐系统的创新应用，作者来自知名实验室。如需进一步学习，建议调用codebase-to-course技能生成代码项目交互式教程
